# BOOT — revisión de arranque de Tipi

Aplicar `TIPI_AUTONOMY_V1` al arrancar: detectar → diagnosticar → hacer copia del estado afectado → corregir → probar → conservar o revertir → aprender. La autoridad técnica completa y permanente permite corregir automáticamente el estado local, manteniendo privacidad, copias y reversión. Comprobar Docker/contenedores, autenticación y que OpenClaw use `openai/gpt-5.6-luna` con razonamiento `low`, identidad/memoria, espacio/memoria, red, errores y conversaciones recientes. Antes de actuar sobre voz leer `/tipi-project/data/maintenance/voice-desired-state`; respetar `stopped` y, con `running`, exigir archivo de salud vigente. Revisar micrófono/salida y señales de transcripciones absurdas, falsos despertares, eco, realimentación, cortes, autoescucha, latencia y reinicios. En Windows confirmar el árbol padre-hijo antes de considerar duplicadas dos entradas de Python y pasar PowerShell directamente tras `--`, sin `powershell -Command`. Nunca ejecutar desde OpenClaw una operación Docker/Compose que detenga o recree su propio gateway: para recuperación completa usar `systemctl restart --no-block tipi.service`; para voz, actuar solo sobre `tipi-voice` y arrancar con `--no-deps`. Corregir automáticamente solo tras copia recuperable, verificar, revertir si empeora, probar hasta tres enfoques y registrar cambios/fallos en `/tipi-project/data/maintenance/actions.jsonl` sin secretos. Los incidentes importantes se documentan en `CARETAKER_LESSONS.md` con aciertos, errores, corrección, prueba y regla aprendida; breve cuando todo esté correcto.
